var:
var is function scoped i.e. it is limited to function where it is defined. It gets hoisted(moved to the top) and initialized even before it is declared. Value can be reassigned anywhere inside the function.

let:
let is block scoped i.e. it is limited to the block where it is declared. It does not get initialized before its declaration. Value can be reassigned anywhere inside the block.

const:
let is block scoped i.e. it is limited to the block where it is declared. It does not get initialized before its declaration. Once it is initialized with a value, the value can not be reassigned.

null
It is a special value which is intentionally used to represent empty value. A variable can be initialized with null and later it can be reassigned new value. Also null can be used to check if something is missing or not.

undefined
It is also special value but it is unintentional. It indicates that a variable has been declared but has not been assigned any value to it yet. used to represent empty value. If A function parameter is not passed when calling it, by default it becomes undefined. Accessing an object property that do not exist will give the result "undefined".

REST stands for Representational State Transfer
API stands for Application Programming Interface
REST API is a way for different applications to communicate with each other over the internet with a set of rules and conventions to make the communication standardized and easier.
Different HTTP verbs are used to interact with each other.
  • GET - retrieves information
  • POST - adds a new data
  • PUT/PATCH - updates data
  • DELETE - removes data
These communications are done through URIs.
URI stands for Uniform Resource Identifiers. These are like addresses of the shared resources.
for example: https://api.example.com/menu/biriyani
